# PRD: Gemini AI 분석 및 전수 수집 (AI Analysis & Full Collection)

## 1. 개요
- **목표:** 수집된 원본 피드(`ts_feeds`)를 Gemini AI로 배치 분석하여 '경제/주식 관련성'을 평가하고, 모든 결과를 효율적으로 저장한다.
- **의존성:** 데이터 스펙은 `00_BASE_data-modeling.md`의 `ts_insights` 테이블 정의를 따른다.

## 2. 사용자 스토리 (시스템 관점)
- "시스템은 API 호출 제한(RPM)을 준수하기 위해 여러 개의 트윗을 한 번에 묶어서 분석을 요청해야 한다."
- "시스템은 분석 결과가 70점 미만이더라도 '기타' 카테고리로 저장하여 데이터 유실을 방지해야 한다."
- "시스템은 AI 응답이 끊기거나 형식이 어긋날 경우를 대비하여 견고한 파싱 로직을 갖춰야 한다."

## 3. 기술적 상세 및 요구사항

### 3.1 분석 워크플로우 (Batch Pipeline)
1.  **데이터 준비:** 아직 분석되지 않은 `ts_feeds` 아이템을 **최대 10개** 가져온다.
2.  **배치 프롬프트 구성:**
    - AI의 오타로 인한 ID 매칭 오류를 방지하기 위해, 원본 UUID 대신 **0부터 시작하는 숫자 인덱스(`ID: 0`, `ID: 1`, ...)**를 번호가 매겨진 리스트 형태로 전달한다.
    - AI에게 각 인덱스에 대응하는 분석 결과를 **JSON 배열(`[]`)** 형태로 응답하도록 강제한다.
3.  **매핑 및 저장:** 
    - AI가 반환한 `feed_id` (**숫자 인덱스**)를 다시 원본 UUID와 매칭한다.
    - 매칭된 데이터를 `ts_insights` 테이블에 한꺼번에 저장한다.

### 3.2 AI 추출 데이터 구조 (Zod 스키마)
AI는 각 피드를 분석하여 다음의 데이터를 추출한다. 상세 필드 타입 및 제약조건은 `00_BASE_data-modeling.md`를 참조한다.

- `feed_id` (number): 원본 피드 리스트의 인덱스
- `relevance_score`, `importance`, `market_type`, `mentioned_stocks`, `is_explicit`, `sectors`, `sentiment_direction`, `sentiment_intensity`, `investment_horizon`, `confidence_level`, `logic_type`, `summary_line`

### 3.3 섹터(Sectors) 및 카테고리 가이드
- **섹터 리스트:** `거시경제`, `정책/정치`, `인공지능`, `반도체`, `2차전지`, `빅테크`, `자동차`, `에너지/원자재`, `로봇`, `바이오`, `증권/금융`, `소비재/유통`, `부동산`, `코인`, `공시/실적`, `방산`, `조선`, `기타(분류외)`, `분석불가(비관련)`
- **기타(분류외):** 투자 관련 정보이나 위 섹터에 해당하지 않는 경우.
- **분석불가(비관련):** 투자와 전혀 무관한 일상 대화, 광고 등.
