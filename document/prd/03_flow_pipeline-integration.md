# PRD: 통합 데이터 파이프라인 및 스케줄러 (Workflow Integration)

## 1. 개요
- **목표:** 전문가의 최신 피드를 수집하고, 수집된 데이터를 Gemini AI로 분석하여 인사이트를 도출하는 전체 과정을 자동화된 파이프라인으로 통합하며, 사용자에게 실시간 진행 상황을 시각적으로 제공한다.
- **주요 작업:** 전체 전문가 순회 수집, 미분석 피드 추출 및 배치 분석 트리거, 실시간 상태 모달 UI 구현, 중단된 로그 관리.

## 2. 사용자 스토리 (시스템 및 관리자 관점)
- "시스템은 주기적으로 모든 전문가의 트위터를 확인하여 새로운 글이 있으면 DB에 가져와야 한다."
- "관리자는 수동 실행 시 현재 '수집 중'인지 '분석 중'인지, 몇 개가 완료되었는지 실시간으로 알 수 있어야 한다."
- "시스템은 수집과 분석 중 오류가 발생하더라도 로그를 남기고 다음 전문가의 데이터를 계속 처리해야 한다."
- "네트워크 단절 등으로 브라우저가 새로고침되어도 진행 중이던 작업의 상태를 다시 확인할 수 있어야 한다."

## 3. 기술적 상세 및 요구사항

### 3.1 통합 프로세스 흐름 (Pipeline Phases)
파이프라인은 다음의 마스터 단계를 거치며 진행된다:
1.  **Phase 1: 시스템 준비 (prep)** - 로그 초기화 및 환경 점검.
2.  **Phase 2: 전문가 피드 수집 (sync)** - 활성화된 전문가별 RSS 동기화.
3.  **Phase 3: AI 인사이트 분석 (analyze)** - 미분석 피드 배치 처리.
4.  **Phase 4: 데이터 정리 및 완료 (done)** - 최종 상태 업데이트 및 리소스 정리.

### 3.2 실시간 진행 상태 시각화 (UI/UX)
- **리스트형 진행 모달**: 사용자가 실행 버튼을 누르면 즉시 `Dialog`가 열리며 4단계(또는 단독 분석 시 3단계) 리스트가 표시된다.
- **상태 하이라이트**: 현재 진행 중인 단계는 강조색(Primary)과 로딩 애니메이션으로 표시하고, 완료된 단계는 체크마크로 표시한다.
- **실행 모드별 동적 UI**:
    - **전체 실행**: 모든 단계 표시.
    - **미분석 분석 전용**: '전문가 피드 수집' 단계를 제외하고 UI 구성.
- **상태 래칭(Latching)**: 비동기 데이터 통신 중 인덱스가 리셋되는 것을 방지하기 위해 도달한 최고 단계를 고정(Latched)하여 보여준다.

### 3.3 예외 처리 및 안정성
- **중단된 로그 관리 (Stale Logs)**: 
    - 파이프라인 시작 시, 과거에 `진행중` 상태로 멈춰있는 로그들을 찾아 `수집 오류` 및 `(중단됨)` 메시지로 일괄 업데이트한다.
- **실시간 메시지 기록**: `ts_pipeline_logs.error_message` 필드에 현재 진행 상황을 텍스트로 기록한다 (예: `피드 수집 중 (2/5)`, `피드 분석 중 (10/40)`).
- **격리된 오류 처리**: 개별 전문가 수집 실패가 전체 분석 파이프라인을 중단시키지 않도록 예외 범위를 격리한다.

## 4. 파이프라인 로그 기록 규격 (ts_pipeline_logs)
- **status**: `진행중` -> `완료`, `수집 오류`, `분석 오류`.
- **collected_count**: 수집 단계에서 새로 추가된 피드 수.
- **analyzed_count**: 분석 단계에서 성공적으로 처리된 인사이트 수 (실시간 업데이트).
- **error_message**: 
    - 실행 중: `현재 단계 설명 (현재/전체)`
    - 종료 시: 오류 상세 내용 또는 `모든 공정 완료`.
