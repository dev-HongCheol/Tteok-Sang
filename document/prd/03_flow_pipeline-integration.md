# PRD: 통합 데이터 파이프라인 및 스케줄러 (Workflow Integration)

## 1. 개요
- **목표:** 전문가의 최신 피드를 수집하고, 수집된 데이터를 Gemini AI로 분석하여 인사이트를 도출하는 전체 과정을 자동화된 파이프라인으로 통합한다.
- **주요 작업:** 전체 전문가 리스트 순회 수집, 미분석 피드 추출 및 배치 분석 트리거, 통합 실행 함수(Main Flow) 구현.

## 2. 사용자 스토리 (시스템 관점)
- "시스템은 주기적으로 모든 전문가의 트위터를 확인하여 새로운 글이 있으면 DB에 가져와야 한다."
- "시스템은 DB에 저장된 피드 중 아직 AI 분석을 거치지 않은 데이터들을 찾아 배치로 분석을 요청해야 한다."
- "시스템은 수집과 분석 중 오류가 발생하더라도 로그를 남기고 다음 전문가의 데이터를 계속 처리해야 한다."

## 3. 기술적 상세 및 요구사항

### 3.1 통합 프로세스 흐름 (Pipeline)
1.  **Phase 1: 피드 수집 (Expert Sync)**
    - `ts_experts` 테이블에서 활성화된 모든 전문가 목록 조회.
    - 각 전문가별로 `syncExpertFeed` 함수를 호출하여 최신 트윗 수집.
2.  **Phase 2: 배치 분석 (Batch Analysis)**
    - `ts_feeds` 테이블에서 `ts_insights`에 연결되지 않은(미분석) 피드들을 조회.
    - 최대 10개씩 묶어서 `analyzeFeedsBatch` 함수 호출.
3.  **Phase 3: 상태 리포트**
    - 총 수집된 피드 수와 성공적으로 분석된 인사이트 수를 요약 출력/기록.

### 3.2 예외 처리 및 안정성
- **개별 전문가 격리:** 한 전문가의 RSS 수집이 실패해도 전체 프로세스가 멈추지 않고 다음 전문가로 넘어가야 함.
- **Rate Limit 준수:** Gemini API 호출 사이에 짧은 지연(예: 1~2초)을 두어 무료 티어 제한을 방어함.
